03 — Featured project

Options Backtesting Dashboard

How I turned billion-row, full-day Excel loads into a strategy-research dashboard that refreshes in seconds.

hours → days became seconds → minutes

~1,000× faster strategy iteration on the same data

● Before

Excel, reloaded from scratch

  • Re-loaded the entire dataset every single time
  • Data runs 100s of millions → 1 billion+ rows
  • Excel caps at ~1M rows — couldn't even open the raw data
  • Full loads took hours, sometimes a whole day
→
● After

Python pipeline + incremental aggregation

  • Pre-aggregates raw data into partitioned Parquet
  • Each run processes only new dates, then appends
  • Billions of rows viewable as aggregated heatmaps
  • Refreshes in seconds to minutes

How it works

01

Ingest

Read RAR archives of raw backtest output and normalize to Parquet with Polars — chunked, memory-managed.

02

Aggregate · incrementally

Track processed dates so each run aggregates only new data, appending to partitioned files. No full reloads.

03

Query

DuckDB runs SQL straight over the Parquet files — group-bys across params, DTE, weekday — in milliseconds.

04

Explore

A Streamlit app with an Excel-style drag-and-drop pivot builder, saved layouts, and formatted Excel heatmaps via xlwings.
